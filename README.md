Жадібні алгоритми та динамічне програмування


____Опис завдання
У даній роботі реалізовано два підходи до задачі видачі решти покупцеві з використанням заданих номіналів монет:  
`[50, 25, 10, 5, 2, 1]`.

Було реалізовано:
- жадібний алгоритм;
- алгоритм динамічного програмування.

Метою роботи є порівняння ефективності цих двох підходів.

---

____Жадібний алгоритм
Функція `find_coins_greedy` працює за принципом вибору найбільшого можливого номіналу монети на кожному кроці.  
Цей підхід є швидким і має лінійну складність відносно кількості номіналів монет — **O(n)**.

Перевагою жадібного алгоритму є висока швидкість роботи, однак він не завжди гарантує мінімальну кількість монет для довільних наборів номіналів.

---

____Алгоритм динамічного програмування
Функція `find_min_coins` використовує метод динамічного програмування для знаходження мінімальної кількості монет для заданої суми.  
Алгоритм будує оптимальні рішення для всіх сум від `0` до заданої, що гарантує оптимальний результат.

Часова складність алгоритму — **O(n × amount)**, що робить його менш ефективним при великих сумах.

---

____Порівняння алгоритмів
Жадібний алгоритм працює значно швидше та підходить для великих сум, проте не завжди є оптимальним у загальному випадку.  
Алгоритм динамічного програмування гарантує мінімальну кількість монет, але потребує більше часу та пам’яті.

Для даного набору монет жадібний алгоритм дає оптимальний результат, тому є доцільнішим у практичних сценаріях.

---

____Висновки
У роботі реалізовано та порівняно два підходи до задачі видачі решти.  
Жадібний алгоритм є більш ефективним за швидкістю, тоді як алгоритм динамічного програмування забезпечує оптимальність результату.  
Вибір алгоритму залежить від вимог до швидкодії та точності у конкретній задачі.
